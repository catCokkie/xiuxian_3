# 00 愿景

## 产品定位
- 类型：桌面底部驻留的修仙桌宠 + 轻量养成。
- 核心驱动：读取用户真实键盘/鼠标操作次数作为修炼进度。
- 引擎与语言：Godot 4 + C#。
- 运行形态：常驻后台，用户工作学习时自然积累进度。

## 核心体验
1. 做事即修仙：用户在日常打字、点击中持续成长。
2. 低打扰陪伴：桌宠在桌面底部活动，不遮挡主工作区。
3. 明确可控：进度规则透明，玩家能通过策略分配资源。

## V1 范围
- 包含：
- 全局键鼠计数与活动点转换。
- 境界成长与突破。
- 桌面底栏展示 + 书卷式系统面板。
- 轻量探索遭遇与按输入推进的战斗回合。
- 最近战斗日志、固定装备奖励、背包/装备的最小闭环。
- 本地存档、基础防刷护栏与统一存档恢复。
- `Bug反馈` 最小闭环（问题描述、复制日志路径、导出反馈文件、打开数据目录）。
- 灵宠好感/亲和等长期资源沉淀。
- 不包含：
- 联网排行/社交。
- 复杂技能战斗、多层状态异常、大地图探索。
- 账号云同步。
- 灵宠心情互动闭环与短时维护玩法。
- 完整装备内容池、随机词条、强化和套装养成。

## 当前实现口径（2026-03-23）
- 当前版本已完成最小可玩闭环：输入推进探索、遭遇战、资源结算、最近战斗日志、固定奖励装备、背包与手动装备。
- 桌面 UI 已形成 `主条 + 图书子菜单` 的稳定结构，子菜单页签收口为 `修炼概况 / 战斗日志 / 装备情况 / 统计概览 / 配置校验 / Bug反馈 / 设置`。
- 设置页只保留当前可感知或已接线选项，未完成能力通过 `（预留）` / `（实验）` 降低误解成本。
- 当前突破为“经验满即可手动突破”的确定性口径，不含突破丹消耗和失败保底机制。
- 配置校验已进入子菜单，支持按范围和当前关卡过滤，并可直接运行掉落模拟；调试入口仍保留快捷键操作。

## 隐私与合规原则
- 只统计事件数量与频率，不记录按键内容。
- 不上传原始输入数据，默认仅本地处理。
- 提供一键暂停采集和状态可视化提示。

## V1 目标指标
- D1 留存 >= 40%。
- 首日 60-90 分钟累计使用可达第 3 境。
- 70% 用户能理解“输入行为 -> 修炼收益”的映射关系。
