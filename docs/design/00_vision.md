# 00 愿景

## 产品定位
- 类型：桌面底部驻留的修仙桌宠挂机游戏 + 轻策略养成。
- 核心驱动：读取用户真实键盘/鼠标操作次数作为修炼进度。
- 引擎与语言：Godot 4 + C#。
- 运行形态：常驻后台，用户工作学习时自然积累进度，并在短时回看时做资源与模式分配。

## 核心体验
1. 做事即修仙：用户在日常打字、点击中持续成长。
2. 低打扰陪伴：桌宠在桌面底部活动，不遮挡主工作区。
3. 明确可控：进度规则透明，玩家能通过主行为切换与资源分配做轻策略决策。
4. 低注意力回看：用户不需要持续盯屏，也能在回看时快速理解“这一轮发生了什么、接下来该做什么”。

## V1 范围
- 包含：
- 全局键鼠计数与活动点转换。
- 境界成长与突破。
- 桌面底栏展示 + 书卷式系统面板。
- 轻量探索遭遇与按输入推进的战斗回合。
- 12 个活动模式组成的基础资源蛛网（采集 / 加工 / 修行）。
- 最近战斗日志、动态回看、节律提醒等低注意力反馈能力。
- 装备、背包、基础加工与常驻增益的最小养成闭环。
- 本地存档、基础防刷护栏与统一存档恢复。
- `Bug反馈` 最小闭环（问题描述、复制日志路径、导出反馈文件、打开数据目录）。
- 灵宠好感/亲和等长期资源沉淀。（V2 预留）
- 不包含：
- 联网排行/社交。
- 复杂技能战斗、多层状态异常、大地图探索。
- 账号云同步。
- 灵宠心情互动闭环与短时维护玩法。
- 完整装备内容池、随机词条、套装养成和高深度 build 构筑。

## 当前实现口径（2026-03-29）
- 当前版本已完成从“最小可玩闭环”向“多系统轻策略挂机”的扩展：输入推进、战斗、资源结算、12 模式主行为、子系统精通、基础装备与加工链路均已落地。
- 产品结构已形成 `主条 + 图书子菜单` 的稳定双层界面，玩家可在低打扰挂机与短时集中管理之间切换。
- 成长方向已从单一“副本/修炼”扩展为 12 系统资源蛛网，但内容厚度、事件回看、消费端压力和部分系统独特性仍在持续打磨中。
- 当前突破为“经验满即可手动突破”的确定性口径；悟性已转为横向子系统成长货币。
- 当前设计重点已从“能否跑通”转向“是否容易理解、是否有切换决策、是否能被玩家清晰感知”。

## 隐私与合规原则
- 只统计事件数量与频率，不记录按键内容。
- 不上传原始输入数据，默认仅本地处理。
- 提供一键暂停采集和状态可视化提示。

## V1 目标指标
- D1 留存 >= 40%。
- 首日 60-90 分钟累计使用可达第 3 境。
- 70% 用户能理解“输入行为 -> 修炼收益”的映射关系。
