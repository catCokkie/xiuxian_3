# 04 开发里程碑

## M0 - 技术预研（2-3 天）
- 完成 Windows 全局键鼠计数 PoC（仅次数，不含内容）。
- 确认 Godot 窗口置底驻留方案与多分辨率适配策略。
- 输出输入采集权限提示与隐私文案。

## M1 - 核心管线（4-6 天）
- 接入 `InputHookService` + AP 聚合。
- 完成 AP -> 资源 -> 境界经验的结算链路。
- 建立本地存档与异常输入保护。
- 规则：探索进度的唯一输入源是 `InputActivityState.InputBatchTick`（按输入事件计数推进；禁止在 `ExploreProgressController` 内做本地 `_Input` 统计）。

## 当前进度快照（2026-02-26）
- `M1-1` 已完成：`InputHookService -> InputActivityState -> ExploreProgressController` 已串联。
- `M1-2` 已完成：输入与状态服务 AutoLoad 已接入（`InputActivityState`、`InputHookService`、`InputPauseShortcut`、`LevelConfigLoader`、`ResourceWalletState`、`PlayerProgressState`、`ActivityConversionService`）。
- `M1-3` 已完成：AP -> 资源 -> 境界经验链路已打通，探索进度按输入事件独立推进。
- `M1-4` 已完成：关卡/怪物/掉落改为 JSON 配置驱动，支持多关卡轮转、战斗结算、保底与上限规则。

## 当前实现补充（2026-03-23）
- 子菜单结构已收口：`修炼概况 / 动态 / 装备情况 / 统计概览 / 配置校验 / Bug反馈 / 设置`。
- 战斗事件已合并至动态 Tab 的"战斗"筛选视图中。
- 战斗主线已拆出 `BattleStartRules / BattleRules / BattleLifecycleRules / RewardRules`。
- 玩家/怪物/装备已进入统一属性管线，装备能真实影响战斗。
- 已支持最小装备玩法闭环：首通固定装备奖励 -> 背包 -> 手动装备 -> 存档恢复。
- `Bug反馈` 已具备问题描述、复制日志路径、导出反馈文件、打开数据目录的最小闭环。
- 设置页已隐藏 demo 无用项，并对 `云端同步`、`任务栏图标`、`游戏缩放比例` 等未完全接线项使用 `（预留）` / `（实验）` 标识。
- `LevelConfigLoader` 校验结果已接入子菜单 `配置校验` 页签，支持范围/当前关卡过滤，并可在子菜单内直接运行掉落模拟；调试入口仍保留快捷键操作。
- 自动化回归、统一测试入口、GitHub Actions 已接入。

## M2 - 桌面 UI（4-5 天）
- 实现底部桌宠条（常驻、低干扰）。
- 实现书卷式系统面板（顶部左右页签组 + 单页全宽内容区）。
- 主条演出区保留核心状态，减少无效说明行。
- UI 文案统一收口到 `scripts/ui/UiText.cs`。

## M2 当前实现口径（与代码对齐）
- 子菜单窗口：顶部左右两组页签，内容区单页全宽显示，左上角关闭按钮。
- 主条窗口：`BattleTrack` 精简为一行核心状态 + 角色/怪物/HP 信息。
- 状态机最小集：`Exploring`、`InBattle`、`ExploreComplete`。
- `battle_state` 存档口径：`exploring` / `in_battle`（读档安全回退到 `exploring`）。

- 灵石经济：产出/消费闭环接入。
- 完成首轮资源产消平衡验算。
- （已移除灵宠数值系统，V1 不含灵宠互动。）

## M5 - 可扩展活动框架 + 悟性熟练度（Phase 6, 5-7 天）
- 实现 `IActivityDefinition` / `IRecipeDefinition` / `ActivityRegistry` / `MaterialRegistry` 可扩展框架。
- 悟性熟练度系统：11 系统 × 4 级 = 1900 悟性总投入，`InsightMasteryState` + `InsightMasteryRules`。
- 存档升级 v7→v8：新增 `mastery.*`、`material_inventory`、`activity_registry` 字段。
- 内容基础：16 种材料 ID 注册、材料产出/消耗关系表。
- 验收：框架可加载新活动定义，悟性可投入解锁熟练度等级。

## M6 - 8 新子系统实现（Phase 7, 8-12 天）
- **采集系新增**：灵田（种植/收获循环）、矿脉（选矿点/挖掘）、灵渔（选钓点/垂钓）。
- **加工系新增**：符箓（碎符+灵墨→符咒）、烹饪（灵鱼+灵果→灵膳）、阵法（灵玉+化石→阵盘）。
- **修行系新增**：体修（兽骨+毒腮→永久属性）。
- 战斗集成：符咒自动消耗、灵膳 Buff、阵盘被动槽位。
- 离线结算：8 新系统离线规则（灵田/符箓/烹饪/阵法支持，其余回退修炼）。
- 11 模式 UI 选择器：分组二级菜单替代旧 4 模式下拉。
- 灵石消费端扩充：种子商店、矿脉刷新、灵塘扩容等 8 项新消费。
- 验收：11 模式全部可切换运行，资源蛛网闭环，产消比健康。

## V1 验收标准
- 用户可在后台使用场景中持续获得可感知进度。
- 桌面底栏稳定驻留，不影响主工作区操作。
- 输入采集全程不记录敏感内容。
- 60-90 分钟累计使用可达到第 3 境。
