# 16 子系统深化意见

> 生成日期：2026-03-30
> 基于：设计文档 00-15 + 代码实现（317 测试全绿、12 子系统落地、存档 v9）
> 用途：逐项对齐后更新到 `10_todo.md` 作为 V1.5 任务来源

---

## 总体判断

项目当前处于"核心引擎已封装、内容尚浅"阶段。深化方向聚焦三个维度：

1. **内容厚度**：配方/装备/怪物数量不足以支撑 2 周以上的新鲜感
2. **玩家感知反馈**：关键事件（突破、掉落、精通解锁）缺少即时反馈
3. **经济平衡**：灵石消费端几乎为空，悟性消费路径单一

## 对齐清单

### D-01 Toast 通知系统
- **优先级**：P0
- **现状**：突破成功、精通解锁、首通装备、炼丹完成等关键事件无即时反馈，TODO 中标记为 V2
- **建议**：提前到 V1.5。实现一个轻量 Toast 队列（屏幕上方居中，自动消失 2-3 秒），覆盖以下事件：
  - 境界突破成功
  - 精通等级解锁
  - 装备掉落（含稀有度着色）
  - 炼丹/炼器/锻造完成
  - 离线结算摘要
- **工作量预估**：新增 `ToastController.cs` + 场景节点，约 80-120 行
- **状态**：✅ 已完成
- **实现记录**：
  - 新增 `scripts/ui/ToastController.cs`（~230 行）：队列式 Toast 通知，屏幕顶部居中，自动消失 2.8 秒 + 0.4 秒淡出，最多同时 3 条
  - 已订阅事件：`RealmLevelUp`（突破）、`MasteryChanged`（精通解锁）、`EquipmentInventoryChanged`（装备掉落）、`AlchemyChanged`（炼丹完成）、`SmithingChanged`（强化完成）、离线结算摘要
  - 场景集成：`PrototypeRoot.tscn` 新增 `ToastController` 节点
  - `PrototypeRootController.cs` 新增 `_toastController` 引用 + 离线结算 Toast
  - 331/331 测试通过，0 编译错误
- **评估（2026-03-30）**：
  作为读取点击的桌宠挂机游戏，纯 Toast 作为唯一事件通知层 **不够合适**。核心问题：
  1. **玩家不在看**：桌宠长期遮盖在其它窗口后面，2.8 秒 Toast 一闪即逝，里程碑事件无人知晓
  2. **无持久历史**：战斗日志仅保留最近 10 条战斗记录，精通解锁 / 炼丹完成 / 突破等事件零持久化
  3. **重复事件刷屏**：连续炼丹出锅、连续掉装备时每次都弹独立 Toast
  4. **AFK 无感知**：60 秒减速→120 秒暂停完全静默
  5. **回归无追溯**：短暂离开（非离线结算）回来后，错过的事件没有回看途径
  → 需要三层通知体系改造，见 D-21 ~ D-24
