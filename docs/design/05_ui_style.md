# 05 界面风格规范（参考图）

## 总体方向
- 视觉关键词：古卷、木框、暖棕、像素灵宠。
- 布局结构：`底部常驻条 + 中上部可展开主面板`。
- 目标：既有桌宠陪伴感，也有 RPG 面板的系统深度感。

## 分层定义
1. 桌面底栏（常驻）
- 高度建议：屏幕高的 10%-14%。
- 内容：灵宠动画、境界进度条、AP 实时速率、快捷按钮。
- 交互：支持拖动排序，默认半透明低打扰。

2. 主系统面板（展开）
- 采用“书页/卷轴”形态，顶部页签分为左右两组：
- 左组（游戏内容）：`修炼概况`、`动态`、`装备情况`、`统计概览`。
- 右组（功能设置）：`Bug反馈`、`设置`。
- 主内容区按当前页签切换内容，使用单页全宽展示（不做左右双栏）。
- 左上角提供显式关闭按钮。
- 底部状态条固定显示灵石数量与货币图标。
- 关闭时回到底栏，不销毁状态。
- `动态` 页签延续书页式阅读感，事件列表按"最新在上"排列，单条记录控制在 2-3 行内，避免信息墙。
- 战斗类事件使用轻量状态色区分：胜利偏暖金、失败/撤退偏灰红，但不使用持续闪烁或高频动效。
- Demo 版优先展示真实可用入口；未接线完成的功能入口不在页签层直接暴露。
- `Bug反馈` 页使用实用型布局：上方说明，中间多行输入，下方为复制/导出/打开目录动作行。
- `设置` 页对非正式功能不隐藏语义，而是显式标注 `（预留）` / `（实验）`，避免玩家把 demo 选项误判为正式承诺。

3. 文案管理
- UI 文字统一收口在 `scripts/ui/UiText.cs`。
- 场景中的文本仅作默认值，运行时由控制器从 `UiText` 覆盖。
- 新增文案时优先加到 `UiText`，避免在多个脚本重复硬编码。

## 美术与动效约束
- 主色：羊皮纸米黄 + 木质棕 + 铜金点缀。
- 字体：像素或手写风标题，正文保持可读性。
- 动效：
- 展开面板用“翻页/滑出”0.2-0.3s。
- 底栏灵宠保持低频呼吸与眨眼动画。
- 状态变化（突破/掉落）给短时高亮，不长时间闪烁。

## 可用性约束
- 任何分辨率下底栏不得遮挡任务栏关键区域。
- 面板默认居中，支持记忆上次位置。
- 提供“工作模式”：隐藏面板，仅保留极简底栏。

## 活动模式选择器规范

### 设计目标
- 从 4 模式升级到 11 模式后，扁平下拉列表过长。改用**分组二级菜单**保持低打扰。

### 视觉结构
```
┌────────────────────┐
│ ▼ 当前：炼丹       │  ← 底栏右下常驻入口（单行显示当前模式）
└────────────────────┘
        ↓ 点击展开
┌────────────────────┐
│ 📦 采集            │  ← 分组标题（不可选）
│   副本 · 灵田      │
│   矿脉 · 灵渔      │
│ ⚒ 加工             │
│   炼丹 · 炼器      │
│   符箓 · 烹饪 · 阵法│
│ 🧘 修行            │
│   修炼 · 体修    │
└────────────────────┘
```

### 样式约束
- 分组标题：铜金色小字 + 分隔线，不可点击。
- 可选模式：淡褐色文本，hover 时背景微亮。
- 未解锁模式：灰色文本 + 锁图标，hover 显示 tooltip `"需要 XX 熟练度 Lv1"`。
- 当前模式：左侧显示金色圆点标记。
- 展开动效：从底栏向上弹出，0.15s 淡入。
- 收起：点击任意模式或点击外部区域。
- 字体与主色沿用羊皮纸米黄 + 木质棕风格。
- 详细交互定义见 `02_systems.md` §21（渐进解锁）和 `15_ui_prompts.md`。

## Toast 通知规范

> **实现状态（2026-03-29）**：Toast 系统在 v1 中尚未实装，`ToastController.cs` 尚未创建。本规范为 V1 后续扩展预留参考。当前关键里程碑提示（突破/首杀等）通过状态栏文案和修炼概况页展示。

### 设计目标
- 桌宠产品以“低打扰”为核心，Toast 必须轻量、短暂、不阻断操作。
- 让玩家在低注意力状态下仍能感知关键里程碑。

### 通知分级
| 级别 | 触发场景 | 持续时间 | 位置 | 样式 |
|------|---------|---------|------|------|
| **重要** | 境界突破成功、Boss 首杀、区域解锁 | 3-4s | 底栏上方居中 | 金色边框 + 微光晕动效 |
| **普通** | 炼丹完成、强化完成、装备掉落 | 2-3s | 底栏上方居中 | 淡褐色背景 |
| **信息** | 丹药自动消耗、日上限命中 | 1.5-2s | 底栏上方居中 | 半透明文本 |

### 规则约束
- 同时最多显示 1 条 Toast；多条排队，不叠加。
- Toast 不可点击、不可交互，纯展示。
- 不在工作模式下显示“普通”和“信息”级 Toast，仅保留“重要”级。
- 动效：淡入 0.2s + 停留 + 淡出 0.3s。
- UI 文案统一从 `UiText.cs` 提供。
- 代码建议：新建 `scripts/ui/ToastController.cs`，提供 `ShowToast(level, message, durationOverride?)` 方法。

## 修行节律提醒规范（周天系统 UI）

### 设计目标
- 将番茄钟式休息提醒映射为"运功周天"世界观，服务于桌宠挂机场景下的节律感和进度回看。
- 提醒本身必须是可选能力，默认不抢焦点、不暂停游戏、不覆盖主工作流。
- 周天完成时给予微量奖励（悟性/灵石），形成正反馈但不构成经济依赖。

### 提醒强度
| 档位 | 触发形式 | 感知强度 | 适用用户 |
|------|---------|---------|---------|
| `无提醒` | 不主动提示，奖励静默发放 | 无 | 希望完全静默挂机的用户 |
| `弱提醒` | Toast + 底栏高亮"小周天圆满" + 可选未读徽标 | 低 | 默认档，适合大多数桌宠用户 |
| `强提醒` | 居中卡片"小周天·第N轮" + 窗口 attention + 未读徽标 | 中 | 明确希望被提醒休息的用户 |

### 展示结构（周天摘要卡片）
- 标题：`小周天·第 N 轮` / `大周天圆满`，使用修仙世界观命名。
- 奖励行："+3 悟性（周天奖励）"以金色 (#C8A050) 高亮显示。
- 主摘要：展示一个周期内的 `灵气`、`悟性`、`境界经验` 主增量。
- 次摘要：展示当前主行为推进，例如 `探索 +18%`、`完成回气丹 2 批`、`收获灵草 6`。
- 事件角标：用一行小字汇总 `突破 1 / 精通解锁 1 / 装备 2` 等关键事件数。
- 建议文案：调息期 → "运功一周天，建议起身调息片刻"；入定期 → "四周天圆满，可入定休憩"。语气中性，不做命令式文案。

### 样式约束
- `弱提醒` 沿用 Toast 体系的暖棕 + 铜金点缀，控制在 2-3 行内，不做居中遮挡。
- `强提醒` 可在主条上方居中出现更完整卡片，但高度不应超过屏幕高的 18%。
- 任何档位都不得使用持续闪烁、频繁抖动或自动抢焦点。
- 若用户处于 `工作模式`，则 `强提醒` 自动降级为 `弱提醒`。
- 调息期底栏可显示淡金色小图标提示"调息中"，不超过 5 分钟后自动消失。

### 配置建议
- 设置页路径：`设置 → 系统 → 修行节律`。
- 选项：开关、提醒强度（三档）、小周天周期（25/45/60/90 分钟）。
- 默认值：`开启 + 弱提醒 + 25 分钟`。
