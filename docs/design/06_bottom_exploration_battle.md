# 06 底部探索/战斗进度界面详细设计

> **唯一定义点原则**：战斗消耗品（丹药/符咒/灵膳/阵盘）的配方和材料定义在 `02_systems.md` §14-§18。本文仅定义战斗中的自动消耗规则和 UI 交互。

## 0. 设计目标
- 目标一句话：让玩家在桌面底部用最低打扰成本持续看到“探索推进与战斗结果”。
- 3 秒可读信息：当前区域、探索进度、当前战斗状态、修炼阶段。
- 与键鼠驱动连接：键鼠输入事件直接推进探索条与遭遇计量条；`activity_point` 用于资源结算。

备注：本文混合记录“当前实现”与“后续设计目标”；若两者冲突，以带有“当前/实装/实现口径”的描述为准。

## 当前产品边界（2026-03）
- 已实现：探索推进、怪物接近触发遭遇、按输入推进战斗回合、战斗日志、首通固定装备奖励、背包与手动装备。
- 原型/预留：完整装备内容池、复杂技能战斗、联机、云同步、灵宠互动闭环。

## 1. 双窗口结构
1. 横向主窗口（常驻底部）
- 作用：展示探索与战斗过程。
- 位置：桌面底部贴边，默认横向铺开。
- 特征：低高度、长条形、像素场景演出。

2. 子菜单窗口（图书窗口）
- 作用：以书页/卷轴形式展示系统菜单，顶部为左右两组页签，内容区单页全宽展示。
- 触发：主窗口左下角“图书按钮”。
- 行为：独立显示/隐藏，不影响主窗口探索计时。

## 1.1 图书窗口结构（固定）
- 窗口样式：摊开书本或卷轴展开，顶部一排页签，底部状态条。
- 顶部分组：
- 左侧页签组（游戏内容）：
- `修炼概况`
- `战斗日志`
- `装备情况`
- `统计概览`
- `配置校验`
- 右侧页签组（功能/设置）：
- `Bug反馈`
- `设置`
- 内容区：
- 选中任意页签后，内容填充同一页内容区（不做左右分栏同时展示）。
- 设置页也在同一内容区内切换子模块。
- `战斗日志` 迁移后，主横向窗口不再新增独立日志面板入口，统一通过左侧页签查看。
- 底部信息条：
- 固定显示 `灵石数量`，右对齐，带货币图标。
- 可选显示次级货币（V1 可不做）。
- 左上角：
- 关闭按钮（`X`），行为与图书按钮一致（切换显隐）。

## 2. 横向主窗口布局（从左到右）
- 左上：`拖动按钮A`
- 用途：拖动主窗口位置（默认仅限桌面底部水平移动；调试模式可自由移动）。
- 左下：`图书按钮`
- 用途：打开/关闭子菜单窗口。
- 图书按钮下方：`修炼阶段`
- 显示：如“炼气三层/筑基初期”。
- 中央横向区域：`探索战斗演出带`
- 显示：角色、敌人、小怪、地形、遭遇提示。
- 实装口径：仅保留一行核心状态文本；演出标题与辅助说明行默认隐藏，减少视觉噪声。
- 右下：`当前区域探索进度条`
- 显示：区域名 + 0-100% 进度。
- 右上：`拖动按钮B（缩放柄）`
- 用途：调整横向主窗口宽度（最小/最大宽限制）。

## 3. 交互定义
- `拖动按钮A`：
- 鼠标按下后进入拖动态，释放时保存窗口位置。
- `图书按钮`：
- 单击切换子菜单窗口显隐。
- 打开时默认落在上次页签；首次打开默认 `修炼概况`。
- `拖动按钮B`：
- 水平方向拖拽修改主窗口宽度。
- 实时重排内部 UI，最右探索条保持可见。

## 3.1 图书页签交互
- 左右页签组分别维护激活项；内容区当前显示由“最近一次点击的页签”决定。
- 切换页签时播放 0.2 秒翻页/滑页动效。
- `Bug反馈` 页签进入后提供：
- 问题描述输入框（多行）
- 一键复制日志路径按钮
- 导出反馈文件按钮（写入本地反馈目录）
- 打开数据目录按钮
- Demo 版隐藏 `联机` 页签入口；后续若进入联网阶段，再作为独立功能页签恢复。
- `设置` 页仅保留当前可感知或已接线的选项；`联机`、`管理员模式`、`手写支持`、`开机启动动画` 等预留项默认隐藏，不作为当前可点击入口。
- `设置` 页中 `任务栏图标`、`云端同步` 等未完整接线选项显式标注 `（预留）`；`游戏缩放比例`、`里程碑提示` 等部分接线项标注 `（实验）`。

## 3.3 装备情况页签（最小可用）
- 当前版本至少展示已装备物品列表，不再只是占位模板。
- 默认空存档会自动注入一套开发/测试 starter loadout，便于直接观察装备对战斗属性的影响。
- 展示信息包含：已装备列表、背包装备列表、核心属性加成，以及装备前/装备后的玩家基础战斗属性对比。
- 当前已提供最小手动装备入口：从背包武器/护具手动装备到对应槽位；若槽位已有装备，旧装备退回背包。
- 新获得装备不会自动替换当前已装备物品，这是当前产品边界的一部分。

## 3.4 配置校验页签
- 入口位置：子菜单左侧页签，位于 `统计概览` 后。
- 当前实现：展示 `LevelConfigLoader` 校验结果，可在子菜单中直接查看，不再只依赖主条调试面板。
- 筛选方式：支持切换 `全部 / 全局 / 关卡 / 怪物 / 掉落` 范围，并支持 `全部关卡 / 当前关卡` 两种视角。
- 模拟能力：支持切换模拟关卡/怪物，并直接运行 `200` / `1000` 次掉落模拟查看摘要结果。
- 展示目标：让设计/调参时能快速定位 `level_id / monster_id / drop_table_id` 关联错误。

## 3.2 战斗日志页签
- 入口位置：子菜单左侧页签第二位，位于 `修炼概况` 后、`装备情况` 前。
- 设计目标：把最近战斗信息从主界面迁移到书页内容区，减少底栏噪声，同时保留可回看性。
- 默认行为：打开子菜单后保持上次左侧页签；首次进入 `战斗日志` 时直接显示最近记录列表。

### 内容结构
- 顶部摘要区：显示 `最近战斗日志` 标题、已记录条数（如 `10/10`）、最近一次战斗时间。
- 主列表区：按时间倒序展示最近 10 条战斗记录。
- 空状态：显示 `当前暂无战斗记录，继续探索后将在此处留下战斗结果。`

### 单条记录字段
- `time_label`：如 `12:48` / `2 分钟前`，V1 优先使用本地时间短格式。
- `zone_name`：战斗发生区域。
- `monster_name`：怪物名称；若为 Boss，可附 `Boss` 标记。
- `battle_result`：`胜利` / `失败` / `撤退`（V1 默认至少支持胜利；失败/撤退口径预留）。
- `reward_summary`：灵气、悟性、关键掉落的摘要，例如 `灵气+36 悟性+4 掉落: 凝气草x1`。
- `meta_summary`：可选展示 `保底触发`、`日上限命中`、`小时上限命中` 等关键标记。

### 展示规则
- 单条记录默认 2 行：第一行展示 `时间 + 区域 + 怪物 + 结果`；第二行展示 `奖励摘要/关键标记`。
- 关键掉落最多展示 2 项，超出后合并为 `其他掉落 xN`。
- 最新一条记录允许轻量高亮一次；重新打开面板后不重复播放强调动效。
- 列表不做分页；V1 固定 10 条，超出时丢弃最旧记录。

### 交互规则
- 点击左侧 `战斗日志` 页签后进入该视图，不影响主窗口探索与战斗推进。
- V1 不要求记录展开详情；保持只读列表即可。
- 当前实现按存档恢复最近 10 条记录，列表以文本摘要方式展示。
- 若后续需要导出/筛选，应作为独立迭代，不在本次基础设计范围内。

## 4. 探索与战斗循环
1. 平时状态：`Exploring`
- 键鼠行为持续转换为探索推进值。
- 进度条增长并周期性触发遭遇判定。

2. 遭遇触发：`InBattle`
- 当怪物队列前排进入触发线后，直接进入战斗。
- 主窗口中央区域切换为战斗状态文本与 HP 信息。
<!-- MELVOR-CHANGE: 新增丹药自动消耗机制，参考 Melvor Idle 战斗消耗品 -->
- 战斗开始时检查背包丹药，按优先级自动消耗：
  - `回气丹`：玩家 HP < 50% 时自动使用，恢复 30% 最大 HP（每场战斗最多 1 次）。
  - `聚灵散`：若持有，战斗开始时自动消耗 1 个，本次战斗灵气掉落 +25%。
- 丹药消耗不需玩家手动操作，体现"低打扰桌宠"设计。
- 消耗记录写入战斗日志的 `meta_summary` 字段。

<!-- SUBSYSTEM-EXPAND: 新增符咒/灵膳/阵盘战斗集成规则 -->
- **符咒自动消耗**（符箓系统产出）：
  - 战斗开始时，从背包中优先级最高的符咒自动消耗 1 张。
  - 效果示例：`火球符` → 战斗首回合对怪物造成固定伤害（ATK×0.5）；`护体符` → 本次战斗 DEF+15%。
  - 每场战斗最多消耗 **2 张**不同类型符咒（1 攻击 + 1 防御）。
  - 若背包无符咒则跳过，不影响战斗流程。
  - 消耗记录写入 `meta_summary`：`符咒: 火球符×1`。
- **灵膳预食增益**（烹饪系统产出）：
  - 灵膳为持续 Buff，战斗前自动检查是否有激活的灵膳效果。
  - 灵膳提供 **3 场战斗**的持续增益（如 HP+15%、全属性+5%、灵气掉落+30%）。
  - 用完后从背包自动消耗下一份同类灵膳（若有）。
  - 同一时间只能激活 **1 种**灵膳效果，新灵膳覆盖旧灵膳。
  - 若无灵膳则无 Buff，不影响战斗流程。
  - Buff 状态写入存档 `consumable.active_meal`：`{meal_id, remaining_fights}`。
- **阵盘被动增益**（阵法系统产出）：
  - 阵盘为常驻装备型被动，从背包中装备到专用**阵盘槽位**（§8.2 新增）。
  - 装备后提供持续被动效果（如 `聚灵阵盘` → 灵气被动恢复+20%；`护体阵盘` → DEF+10%）。
  - 阵盘不消耗，无耐久，切换即生效。
  - 同一时间只能装备 **1 个**阵盘（Lv3 熟练度解锁第 2 槽位）。
  - 阵盘效果在战斗和非战斗期间均生效。

3. 战斗结束：`BackToExploring`
- 完成掉落与结算后立即回到探索。
- 怪物队列重排，继续推进本区域。

<!-- REVIEW-DONE: 战斗失败后果已定义 -->
### 3a. 战斗失败后果（Battle Defeat Consequences）
- **普通怪战斗失败**：
  - 不重置探索进度。
  - 不扣除任何资源。
  - 仅跳过本次掉落结算（等同本次战斗无收益）。
  - 战斗日志记录为“失败”，包含怪物名和剩余 HP。
  - 设计意图：桌宠产品不应惩罚玩家，只是“本次你没获得奖励”。
- **精英怪战斗失败**：
  - 同上，不惩罚。
- **Boss 战斗失败**（参见 §5）：
  - 探索进度归零，重新开始本区域循环（自然时间惩罚，不额外扣除资源）。
  - 回到 `Exploring` 状态，从 0% 重新推进。
- **代码对齐**：`BattleDefeatDecision` 中 Boss 战斗时 `ShouldResetExploreProgress = true`，普通/精英战斗 `= false`。`ShouldResetLevel` 始终为 `false`。

4. 区域完成：`ExploreComplete`
- 进度达到 100%，结算区域奖励并切换下一区域。

<!-- MELVOR-CHANGE: 新增 Boss 挑战阶段，参考 Melvor 但采用进度+Boss 模型而非波次链 -->
5. Boss 挑战阶段：`BossChallenge`
- 触发条件：区域探索进度达到 100% 后，不立即切换下一区域，而是进入 Boss 挑战。
- Boss 特征：
  - 每个区域有 1 个 Boss，属性为该区域精英怪的 2-3 倍。
  - Boss 有独立掉落表，保底至少 1 件 `法器` 级以上装备。
  - Boss 战斗回合数上限为 20 回合（超时判负）。
- Boss 战斗流程：
  - 进入 `BossChallenge` 后，底栏显示 Boss 名称、HP 条、特殊标记。
  - 玩家继续通过键鼠输入推进战斗回合。
  - 胜利：结算 Boss 掉落 + 区域完成奖励，解锁下一区域，进度归零进入下一循环。
  - 失败：探索进度归零，重新开始本区域循环（自然时间惩罚，不额外扣除资源）。
- Boss 掉落设计：
  - 首次击杀：保底 1 件 `灵器` + 大量灵气/悟性 + 灵石。
  - 重复击杀：降为普通 Boss 掉落表（`法器` 为主）。
  - Boss 独有材料：用于高阶炼丹/炼器配方（V2 预留）。
  - 悟性奖励用于积累子系统熟练度解锁资本。

## 5. 显示字段与刷新频率
- `realm_stage_text`：修炼阶段，事件触发刷新。
- `zone_name`：当前区域名，区域切换刷新。
- `explore_progress`：0-100%，每 0.2 秒刷新。
- `battle_state`：未遭遇/预警/战斗中/结算中，状态变化刷新（主条只显示核心状态行）。
- `activity_rate`（可选小字）：每秒 AP，1 秒刷新。

## 6. 状态机
1. `Exploring`
- 进入：开始探索或战斗结算后。
- 可见：全底栏 + 探索条。
- 退出：遭遇触发或区域完成。

2. `InBattle`
- 进入：前排怪物进入触发线。
- 可见：敌我血条、攻击动画、结果倾向。
<!-- MELVOR-CHANGE: InBattle 状态新增丹药自动消耗逻辑 -->
- 附加行为：进入时自动检查 `ConsumableUsageRules`，消耗适用丹药。
- 退出：胜负已定。

3. `ExploreComplete`
- 进入：当前区域探索 100%。
<!-- MELVOR-CHANGE: ExploreComplete 不再直接切下一区域，而是进入 BossChallenge -->
- 可见：区域完成提示，显示"Boss 挑战即将开始"。
- 退出：自动进入 `BossChallenge`。

<!-- MELVOR-CHANGE: 新增 BossChallenge 状态 -->
4. `BossChallenge`
- 进入：区域探索 100% 后自动触发。
- 可见：Boss 名称、Boss HP 条、特殊战斗界面。
- 附加行为：Boss 战斗回合上限 20 回合，超时判负。
- 退出（胜利）：结算 Boss 掉落 + 区域奖励 → 探索进度归零 → `Exploring`（开始下一循环）。
- 退出（失败）：探索进度归零 → `Exploring`（重新开始本区域循环）。

备注（实现口径）：
- 当前代码不单独持久化 `Idle / EncounterWarning / BattleResult` 三个状态。
- 存档字段 `battle_state` 仅使用 `exploring` / `in_battle`，并在读档时安全回退为 `exploring`。

## 7. 数值口径（V1）
- 每 100 次键鼠操作 -> 探索进度约 `2.0%`（基础区）。
- `explore_progress_gain = input_events * 0.02% * zone_speed_factor * battle_pause_factor`
- 遭遇判定：每累计 `20%` 探索进度执行一次基础判定。
- 基础遭遇概率：`18% + danger_level * 4%`。
- 单次战斗平均收益：
- `lingqi` 20-60
- `insight` 3-12
- 材料掉落 0-2 个

## 8.1 底栏活动模式显示

底栏右下区域新增**分组下拉选择器**（`OptionButton` 二级菜单），12 模式按采集/加工/修行三组排列：

### 分组结构
| 分组 | 模式 | 底栏中央区域显示 | 右下进度条含义 | 右下状态文本示例 |
|------|------|------------------|----------------|------------------|

### 交互规则
- 一级点击展开分组视图（采集/加工/修行三行），二级点击选择具体模式。
- 选择后立即切换，无确认弹窗。
- 切换不中断 AP 结算 — 灵气/悟性始终按 AP 公式产出。
- 切换只改变"输入事件驱动什么进度"。
- 未解锁模式灰显并显示 tooltip `"需要熟练度 Lv1 解锁"`。
- 默认模式（新存档）：`副本`。

## 11. 新手引导流程（First-Time Experience）

### 设计目标
- 70% 用户在 3 分钟内理解“键鼠输入 → 修炼进度”映射。
- 不使用强制弹窗教程，用“渐进式提示”配合桶宠低打扰设计。

### 引导步骤
1. **首次启动**（无存档）：
   - 底栏显示欢迎文案：`“灵宠已开始感应你的活动…正常工作即可修炼”`，持续 5s 后淡出。
   - 探索进度条旁边显示脉冲箭头动效，引导视线到进度条。
2. **首次探索推进达 10%**：
   - Toast 提示：`“探索进度推进中…继续工作即可”`。
3. **首次触发战斗**：
   - Toast 提示：`“遇到了怪物！继续输入推进战斗回合”`。
4. **首次战斗胜利**：
   - 重要级 Toast：`“战斗胜利！获得灵气 +28”`。
   - 图书按钮显示脉冲提示——引导玩家打开子菜单查看详情。
5. **首次打开子菜单**：
   - 默认落在“修炼概况”页签。
   - 顶部显示一行说明：`“这里可以查看你的修炼进度和资源”`，查看 1 次后永不再显示。

### 技术约束
- 引导状态存入存档 `guide.steps_completed[]`，避免重复触发。
- 引导提示复用 Toast 系统，不新建独立 UI。

> **实现状态（2026-03-29）**：新手引导流程与 Toast 系统均为 V1 后续扩展预留，当前 v1 尚未实装。首次启动的提示暂通过修炼概况页的状态文案承载。
