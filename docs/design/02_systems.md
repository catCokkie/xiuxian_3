
# 02 系统设计

1. 输入采集系统（Keyboard/Mouse Activity Collector）

7. 灵宠成长预留系统

## 1) 输入采集系统
- 目标：稳定统计键鼠活跃度，作为核心进度来源。
- 采集项（V1）：
- `key_down_count`
- `mouse_click_count`
- `mouse_scroll_step`
- `mouse_move_distance_px`（仅累积距离）
- 关键规则：
- 不记录键值文本、不记录窗口标题。
- 仅统计计数与强度，按 1 秒时间片聚合。
- 超高频输入触发衰减，降低脚本刷分收益。
- 平台策略：
- Windows 优先，使用系统级 Hook（Godot C# 调用 Win32）。
- 非 Windows 平台在 V1 可降级为“仅应用内输入计数”。

## 2) 活动点与资源转换
- 中间资源：`activity_point`（AP）。
- 产出资源：`lingqi`、`insight`、`pet_affinity`。
- 设计原则：
- 转换公式公开可见。
- 存在分钟产出软上限，防止极端输入破坏节奏。
- 允许短时技能提高转化率，但不超过上限阈值。

## 3) 境界与突破
- 字段：
- `realm_level` (int)
- `realm_exp` (double)
- `insight` (double)
- 规则：
- 当前实现口径：`realm_exp` 满后可直接手动突破，不消耗突破丹，也无失败概率。
- `breakthrough_pill`、失败保底、成功率提升仍属于后续扩展设计，暂未进入当前代码主闭环。
<!-- REVIEW-FIX-P1: 突破丹 — V1 取消，V2 预留 -->
- **突破丹处理方案**：
  - V1 决策：取消 `novice_breakthrough_pill`，从掉落表中移除。腾出的掉落 slot 改为灵石/灵草，强化资源循环。
  - V2 预留：若 V2 引入突破失败概率或多阶段突破，突破丹可作为"保底道具"重新上线，届时设计目的明确。
  - V1 无代码影响。

## 4) 探索 / 遭遇 / 战斗规则
- 规则层已拆分为：`BattleStartRules`、`BattleRules`、`BattleLifecycleRules`、`RewardRules`。
- 当前主流程口径：
- 怪物在探索轨道上接近主角。
- 进入触发线后进入战斗态。
- 玩家继续通过输入数推进战斗回合。
- 胜负、奖励、Boss 解锁与战斗日志均通过规则层决策后再由控制器应用。

## 5) 装备、背包与最小奖励闭环
- 已具备 `BackpackState`、`EquippedItemsState`、`EquipmentStatProfile` 的最小闭环。
- 当前装备来源：空存档 starter loadout + 首通固定装备奖励。
- 关键产品边界：
- 新装备进入背包，不自动替换已装备物品。
- 仅在玩家手动触发装备动作时才替换同槽位旧装备。
- 当前仍属原型/开发验证：未实现完整装备内容池、随机词条、强化和最终换装 UX。

## 6) 双窗口桌面 UI 系统
- 主横向窗口：
- 常驻桌面底部，承载探索与战斗进度演出。
- 左上按钮用于拖动窗口位置。
- 右上按钮用于调整主窗口横向宽度。
- 左下图书按钮用于打开/关闭子菜单窗口。
- 图书按钮下方展示当前修炼阶段。
- 右下展示当前区域探索进度条。
- 子菜单窗口：
- 独立窗口，采用书页/卷轴形态，顶部为左右两组页签。
- 左侧页签组（游戏内容）：`修炼概况`、`战斗日志`、`装备情况`、`背包`、`统计概览`、`配置校验`。
- 右侧页签组（功能设置）：`Bug反馈`、`设置`。
- 页签内容区采用单页全宽展示（不再左右分栏同时显示）。
- 左上角提供关闭按钮（`X`）。
- 底部状态条固定显示 `灵石数量`。
- 可单独开关，不中断主窗口探索循环。
- `战斗日志` 承载最近战斗记录，不再额外占用主横向窗口常驻区域。
- 战斗日志默认保留最近 10 条，按时间倒序展示，支持显示怪物、结果、掉落与资源结算摘要。
- Demo 阶段隐藏 `联机`、`管理员模式`、`手写支持`、`开机启动动画` 等未形成闭环的入口，避免把预留项暴露为可用功能。
- `Bug反馈` 提供最小闭环：问题描述输入、复制日志路径、导出反馈文件、打开数据目录。
- `配置校验` 页签展示 `LevelConfigLoader` 的校验结果，支持按范围与当前关卡过滤查看，并可直接执行 200/1000 次掉落模拟。
- 设置页对未完全接线的选项使用 `（预留）` / `（实验）` 标识，降低误解成本。
- 交互约束：
- 默认低干扰，面板状态与窗口尺寸需持久化。
- 窗口缩放后核心信息（修炼阶段、探索进度）必须可见。
- UI 文案统一从 `scripts/ui/UiText.cs` 提供，避免分散硬编码与乱码问题。

## 7) 灵宠成长预留
- 当前 V1 不将灵宠系统纳入主成长闭环；完整的亲密度等级、互动/羁绊设计已整体延后到 V2。
- `pet_affinity` 在 V1 中不承担等级成长、全局增益或主动培养职责，最多作为轻量累计统计存在。
- `pet_mood` 不要求玩家维护，也不作为主 UI 重点展示项；但当前代码中仍会影响资源转化倍率，其后续定位待 V2 与灵宠互动/羁绊玩法一起重构。
- 若后续开启灵宠互动玩法，再补充心情恢复、衰减、羁绊被动等完整闭环。

## 8) 存档与防刷
- 存档结构：单文件 `user://save_state.cfg`（ConfigFile）+ `meta.version`。
- 必要字段：
- `meta.version`
- `meta.last_saved_unix`
- `ui.main_bar_x`
- `ui.main_bar_width`
- `ui.submenu_visible`
- `ui.submenu_active_left_tab`
- `ui.submenu_active_right_tab`
- `combat.recent_logs`
- `equipment.equipped`
- `inventory.__equipment_profiles`
- `input.stats`
- `input.hook_paused`
- 迁移策略：
- 优先读取统一存档；若不存在则自动迁移旧 `ui_state.cfg` + `game_state.cfg`。
- 防刷策略：
- 检测异常高频峰值并打标。
- 本地时间跳变超阈值时进入保守结算。
- 长时间无焦点但高输入速率时降低收益权重。

备注（当前状态）：上述防刷目标已在设计中明确；当前规则层已落实高频衰减、掉落 soft-cap / daily-cap 等基础护栏，异常时间跳变与长时间无焦点场景仍需要后续专项补齐与验收。

## 9) 炼丹系统（Alchemy / Potion Crafting）
- 设计灵感：参考 Melvor Idle 的 Herblore，将副本掉落的草药材料转化为战斗消耗品。
- 定位：资源消费端 — 消耗探索掉落的 `灵草` 类材料 + `灵气`，产出战斗可用的丹药。
- 活动模式：玩家在底栏切换到"炼丹"模式后，键鼠输入转化为**炼丹进度**而非探索进度。
- 核心机制：
  - 选择配方 → 消耗材料 → 输入驱动进度条 → 完成后产出丹药进入背包。
  - 每批次需要固定输入量（如 200 次键鼠事件 = 1 批丹药）。
  - AP 仍正常结算灵气/悟性，但探索不推进。
- V1 丹药列表：
- `回气丹`：战斗中 HP < 50% 时自动使用，恢复 30% 最大 HP。消耗：`spirit_herb` ×2 + 灵气 ×50。
- `聚灵散`：下次战斗灵气掉落 +25%。消耗：`spirit_herb` ×3 + 灵气 ×80。
- `破境丹`（后续）：突破时经验需求 -10%。消耗：高阶草药 + 灵气 ×200。
- 丹药使用规则：
  - 战斗消耗品在进入战斗时自动检查并消耗，不需手动操作。
  - 背包中同类丹药堆叠显示数量。
- 存档字段：`alchemy.selected_recipe`, `alchemy.progress`, `backpack.potions[]`。
- 离线支持：当前实现中炼丹模式不会离线推进配方进度；离线仍按修炼式 AP 结算处理，若后续需要真实离线炼丹产出需单独扩展。

## 10) 炼器 / 强化系统（Smithing / Equipment Enhancement）
- 设计灵感：参考 Melvor Idle 的 Smithing，将副本掉落的碎片/碎符材料转化为装备强化。
- 定位：资源消费端 — 消耗 `碎片`、`碎符` 类材料 + `灵气`，提升已有装备的 `EnhanceLevel`。
- 活动模式：玩家在底栏切换到"炼器"模式后，键鼠输入转化为**强化进度**。
- 核心机制：
  - 选择目标装备 → 选择强化配方 → 消耗材料 → 输入驱动进度条 → 完成后 `EnhanceLevel + 1`。
  - 强化上限 = `稀有度等级 × 3`（俗器 3 级，法器 6 级，灵器 9 级，宝器 12 级）。
  - 每级强化提升装备基础属性 8%（乘算叠加）。
  - 每批次需要输入量随强化等级递增（100 + EnhanceLevel × 50 次键鼠事件）。
- V1 强化材料需求：
  - +1~+3：碎片 ×3 + 灵气 ×30
  - +4~+6：碎片 ×5 + 碎符 ×1 + 灵气 ×80
  - +7~+9：碎片 ×8 + 碎符 ×3 + 灵气 ×150
- 存档字段：`smithing.target_equipment_id`, `smithing.progress`。
- 离线支持：当前实现中炼器模式不会离线推进强化进度；离线仍按修炼式 AP 结算处理，若后续需要真实离线炼器产出需单独扩展。
- 注意：`EquipmentInstanceData.EnhanceLevel` 字段已存在于代码中但未消费，此系统将其激活。

## 11) 灵石经济系统（Spirit Stone Economy）
- **问题背景**：灵石显示与产出链路现已接入，但消费端仍明显弱于产出端，尚未形成完整货币闭环。
- 定位：灵石是**通用中间货币**，连接掉落端和消费端。
- 产出来源：
  - 战斗胜利基础灵石奖励（每场 5-15 灵石，按区域缩放）。
- 出售多余材料（设计预留，当前未形成完整 UI/交互闭环）。
  - 首通奖励（60-200 灵石，按区域递增）。
  - Boss 首杀奖励（300-500 灵石）。
- 消费去向：
- 区域材料兑换配方（购买指定部位基础装备，50-200 灵石）。当前仍以设计预留为主，未形成完整玩家消费入口。
- Boss 重挑快速刷新（非首次可用灵石跳过冷却，50 灵石/次）。**V1.x 预留**。
  - 炼丹/炼器加速（灵石替代部分材料缺口，保守定价防止 P2W 感）。**V2 预留**。
- 丹药急购（背包无回气丹时可花 30 灵石直购 1 个）。**V1.x 预留**。
- 出售材料约束：
- 若后续补上出售操作，需弹窗确认并显示出售数量与获得灵石数。
- 当前代码尚无 `BackpackState.SellItem()` 与对应确认弹窗实现。
- 存档字段：`wallet.spirit_stones`。
- 代码影响：
  - `ResourceWalletState` 新增 `SpiritStones` (int) 字段及 `AddSpiritStones()` / `SpendSpiritStones()` 方法。
  - `RewardRules.BuildBattleRewardSummary()` 新增灵石显示。
  - 当前版本仍缺少稳定的灵石消费调用点，后续需补齐出售、兑换或快捷消费入口。

## 资源循环总览（Melvor-Inspired）

```
┌─────────────┐     灵草/材料      ┌─────────────┐
│   副本探索   │ ────────────────→ │   炼丹系统   │
│ (Dungeon)   │     碎片/碎符      │ (Alchemy)   │
│             │ ────────────────→  │             │
│  产出：灵草  │                    │  消耗：灵草  │
│  碎片/碎符   │                    │  灵气        │
│  灵气/悟性   │                    │  产出：丹药   │
└──────┬──────┘                    └──────┬──────┘
       │                                  │
       │  灵气/悟性                        │ 丹药(自动消耗)
       ▼                                  ▼
┌─────────────┐                    ┌─────────────┐
│   打坐修炼   │                    │  战斗(副本内) │
│(Cultivation) │                    │  自动使用丹药 │
│             │                    │  提升生存率   │
│  产出：灵气  │◄───── 灵气 ───────│             │
│  悟性(高效)  │                    └─────────────┘
│  境界经验    │
└──────┬──────┘
       │  灵气
       ▼
┌─────────────┐
│  炼器/强化   │
│ (Smithing)  │
│             │
│  消耗：碎片  │
│  碎符 + 灵气 │
│  产出：装备   │
│  EnhanceLevel│
│  提升        │
└─────────────┘
```

循环说明：
- **副本**产出所有原材料（灵草、碎片、碎符）和基础灵气。
- **打坐修炼**是灵气/悟性的高效产出模式，为炼丹和炼器提供灵气。
- **炼丹**消耗灵草 + 灵气，产出丹药 → 反馈回战斗提升生存率。
- **炼器/强化**消耗碎片/碎符 + 灵气，提升装备强化等级 → 反馈回战斗提升战力。
- 4 个模式形成闭环：**副本产材料 → 炼丹/炼器消耗材料 → 强化战力 → 推进更高副本**。
